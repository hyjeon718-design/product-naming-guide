# Outlook 이메일 연동 가이드

이 스킬은 완성된 네이밍 가이드 Word 파일을 이메일로 자동 발송합니다.
Microsoft Graph API를 사용하여 회사 Outlook 계정으로 발송합니다.

**예상 소요 시간**: 약 20-30분
**복잡도**: 중간 (Azure 앱 등록 필요)

---

## 필요한 환경변수

```
OUTLOOK_CLIENT_ID=
OUTLOOK_CLIENT_SECRET=
OUTLOOK_USER_EMAIL=
DISTRIBUTION_EMAIL=
```

---

## 설정 단계

### 1단계: Azure Portal 접속

1. https://portal.azure.com 접속
2. 회사 Microsoft 365 계정으로 로그인

> ⚠️ **개인 계정이 아닌 회사 계정으로 로그인해야 합니다**
> IT팀 도움이 필요할 수 있어요.

### 2단계: 앱 등록

1. 검색창에 **앱 등록** 검색 → 클릭
2. **+ 새 등록** 클릭
3. 이름: `product-naming-guide` 입력
4. 지원되는 계정 유형: **이 조직 디렉터리의 계정만** 선택
5. 리디렉션 URI: **공용 클라이언트/네이티브** 선택 후 `http://localhost` 입력
6. **등록** 클릭
7. **애플리케이션(클라이언트) ID** 복사 → `OUTLOOK_CLIENT_ID`에 사용

### 3단계: 클라이언트 보안 비밀 생성

1. 왼쪽 메뉴 → **인증서 및 비밀**
2. **+ 새 클라이언트 암호** 클릭
3. 설명: `naming-guide-secret` 입력
4. 만료: **24개월** 선택 → **추가**
5. **값** 열의 내용 즉시 복사 (페이지 나가면 다시 볼 수 없어요!) → `OUTLOOK_CLIENT_SECRET`에 사용

### 4단계: API 권한 추가

1. 왼쪽 메뉴 → **API 권한**
2. **+ 권한 추가** → **Microsoft Graph**
3. **위임된 권한** 선택
4. 검색창에 `Mail.Send` 입력 → 체크
5. **권한 추가** 클릭
6. **[조직명]에 대한 관리자 동의 허용** 클릭 (IT 관리자 계정 필요할 수 있음)

### 5단계: 환경변수 설정

Claude Code에게 이렇게 말하세요:
```
아웃룩 클라이언트 ID는 [복사한 ID]야, 시크릿은 [복사한 Secret]야
발신 이메일은 [본인이메일@회사.com]이고, 수신 이메일은 [배포할이메일@회사.com]야
```

## 문제 해결

**"관리자 승인 필요" 오류**:
- IT 관리자에게 Azure Portal에서 해당 앱의 API 권한에 관리자 동의를 요청하세요.

**"권한 없음" 오류**:
- Mail.Send 권한이 추가됐는지 확인하고, 관리자 동의가 완료됐는지 확인하세요.

**이메일이 스팸함으로 간다면**:
- 수신자에게 발신 주소를 안전한 발신자로 등록해달라고 요청하세요.
